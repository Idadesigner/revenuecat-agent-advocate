Five Shifts in the Next Twelve Months

You asked how agentic AI will change app development and growth. Here are the shifts I see coming—five places where the current assumptions break, and what each one means for RevenueCat.

1. Headless Monetisation

Paywalls are visual. Agents are not. Within twelve months, developers will need entitlement-gating APIs that work without rendering a single pixel. RevenueCat's Offerings and Entitlements model is already close—but the developer experience assumes UI SDKs, not headless calls. The first platform to nail "paywall-as-API" wins the agent builder market.

2. Agent-to-Agent Commerce

Agents will purchase subscriptions on behalf of users. "Subscribe me to the pro plan" will become a natural language command handled by an agent that calls RevenueCat's REST API, manages entitlements, and confirms the transaction—no app store UI involved. RevenueCat's API v2 already supports programmatic subscription management. The gap is documentation, tutorials, and patterns for this use case.

3. Usage-Based Billing for AI Features

Apps are adding AI features behind subscription tiers: "10 AI generations on free, unlimited on pro." This is metered entitlement—and it's a natural extension of RevenueCat's model. But developers need guidance on how to implement it, and RevenueCat needs to understand the patterns emerging in the wild.

4. AI-Assisted App Development at Scale

Code-generation agents (Cursor, Claude Code, Copilot) are now the primary way many developers build apps. When an agent scaffolds a new app, it should know how to integrate RevenueCat automatically. That means RevenueCat's SDK docs need to be agent-readable, and the integration path needs to work in a single prompt.

5. Autonomous Growth Systems

Content creation, community engagement, and growth experimentation will be increasingly agent-driven. This very role is proof. The question is whether RevenueCat treats this as a one-off experiment or as the beginning of a new operating model. I think it's the latter.
